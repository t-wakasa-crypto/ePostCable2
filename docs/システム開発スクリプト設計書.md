# システム開発スクリプト設計書（scripts/）

> 本書は [scripts/CLAUDE.md](../scripts/CLAUDE.md) から参照される詳細仕様書
> である。`scripts/CLAUDE.md` は薄いポインタとして維持し、スクリプト一覧・
> 追加修正のルールはすべて本書に記載する。**スクリプトの追加・修正を行った
> 場合は、必ず本書と `scripts/help.sh` の一覧を同時に更新すること**
> （`scripts/CLAUDE.md` 側は参照先が変わらない限り更新不要）。

開発作業を補助するシェルスクリプト群です。ファイル名は `<カテゴリ>-<操作>.sh` の形式で、
プレフィックスで対象（docker / artisan / composer / npm / git）が分かるようになっています。

各スクリプトは内部で `cd "$(dirname "$0")/.."`（リポジトリルート）に移動してから実行されるため、
どのディレクトリから呼んでも動作します。スクリプト一覧は `./scripts/help.sh` でも確認できます。

## スクリプト一覧

### コンテナ

| スクリプト | 説明 |
| --- | --- |
| `docker-up.sh` | コンテナ起動 (`docker compose up -d`) |
| `docker-down.sh` | コンテナ停止 (`docker compose down`) |
| `docker-setup-fonts.sh` | 日本語フォント (IPAex) をコンテナにインストール（dompdf フォントキャッシュはレンダリング時に自動生成されるため手動登録は不要） |

### データベース

| スクリプト | 説明 |
| --- | --- |
| `artisan-migrate.sh` | マイグレーション実行 |
| `artisan-fresh.sh` | DB初期化＋シード (`migrate:fresh --seed`) |

### 開発ツール

| スクリプト | 説明 |
| --- | --- |
| `artisan-test.sh` | テスト実行（引数可: `./artisan-test.sh --filter ClassName`） |
| `artisan-routes.sh` | ルート一覧（引数可: `./artisan-routes.sh --path admin`） |
| `artisan-modules.sh` | モジュール一覧 |
| `artisan-list.sh` | artisanコマンド一覧（引数で namespace 絞り込み可: `./artisan-list.sh queue`） |
| `artisan-schedule.sh` | 定期実行バッチの一覧・実行タイミング確認 (`schedule:list`) |
| `artisan-clear.sh` | キャッシュクリア (`optimize:clear`) |
| `artisan-pint.sh` | コードフォーマット (Laravel Pint) |
| `composer-autoload.sh` | オートローダー更新 (`composer dump-autoload -o`) |
| `composer-update.sh` | Composerパッケージ更新（引数可: `./composer-update.sh vendor/package`） |

### フロントエンド

| スクリプト | 説明 |
| --- | --- |
| `node-setup.sh` | WSL初回セットアップ: nvm + Node.js + npm install |
| `npm-dev.sh` | Viteウォッチモード (`npm run dev`) |
| `npm-build.sh` | Vite本番ビルド (`npm run build`) |

### Git

| スクリプト | 説明 |
| --- | --- |
| `git-setup.sh` | ローカルセットアップ（クローン後に一度だけ実行: git hooks を設定） |
| `git-status.sh` | 作業状態確認 |
| `git-log.sh` | コミット履歴（グラフ表示、デフォルト20件） |
| `git-branch.sh` | ブランチ一覧 |
| `git-switch.sh` | ブランチ切り替え: `./git-switch.sh <ブランチ名>` |
| `git-new.sh` | 新しいブランチ作成: `./git-new.sh <ブランチ名>` |
| `git-pull.sh` | 現在のブランチをpull |
| `git-push.sh` | 現在のブランチをpush |
| `git-commit.sh` | 対話的ステージ＋コミット: `./git-commit.sh "メッセージ"` |
| `git-diff.sh` | 差分表示（引数可: `./git-diff.sh HEAD~1`） |
| `git-merge.sh` | 現在のブランチをdevelopにマージ（引数可: `./git-merge.sh <ブランチ名>`） |
| `git-branch-delete.sh` | ブランチ削除（ローカル＋リモート）: `./git-branch-delete.sh <ブランチ名>` |

### その他

| スクリプト | 説明 |
| --- | --- |
| `setup.sh` | WSL 初回セットアップ（git hooks・.env コピー・Docker 起動・Composer インストール・DB 初期化・フォント設定・Node セットアップ・フロントビルドを一括実行） |
| `podman-setup.sh` | AlmaLinux 向け Podman セットアップ（podman / podman-docker / podman-compose インストール・ソケット有効化）。AlmaLinux 以外はスキップ |
| `help.sh` | 使用可能なスクリプト一覧を表示 |

## スクリプト追加・修正のルール

- よく使うコマンドや繰り返す操作に気づいたら、新しいスクリプトとしての追加を提案すること。
- 既存スクリプトで不足・改善点があれば、内容修正を提案すること。
- 命名は `<カテゴリ>-<操作>.sh` の形式に揃える。
- 新規作成・修正したら、本書と `help.sh` の一覧も必ず更新する。
