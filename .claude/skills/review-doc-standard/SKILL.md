# 利用者向けレビュー資料 品質基準

## 基本方針
- 技術用語（API・DB・フレームワーク名など）を使わない
- 「システムが何をしてくれるか」を利用者視点で記述する
- 図・表を活用して視覚的にわかりやすくする
- A4 換算で5〜10ページ相当の分量を目安とする

## 利用シナリオの書き方
「〇〇さんが△△したいとき、□□の手順で操作する」形式で記述する。

例：
> 店舗スタッフが在庫を確認したいとき、
> 1. トップ画面から「在庫管理」を選択する
> 2. 商品名または商品コードで検索する
> 3. 現在の在庫数と入庫予定日が表示される

## 承認サインオフ欄の形式
```markdown
## 承認

| 役割 | 氏名 | 確認日 | サイン |
|---|---|---|---|
| 発注者 | | | |
| 業務担当者 | | | |
| システム担当者 | | | |
```

## 品質チェックリスト
- [ ] IT 知識がない人が読んで理解できるか
- [ ] 主要機能がすべて利用シナリオに含まれているか
- [ ] 制約・できないことが明記されているか
- [ ] 未確認事項が残っている場合はその回答を求める形になっているか
