# コードレビュー 品質基準

## 総合評価の判定基準

| 評価 | 基準 |
|---|---|
| ✅ OK | 重大な指摘なし。軽微な改善提案のみ |
| ⚠️ 要修正 | 軽微〜中程度の問題あり。修正後に再確認不要 |
| ❌ 差し戻し | 設計との重大な乖離・セキュリティ脆弱性・未実装の必須機能あり |

## 設計適合性チェック観点

- API のエンドポイント・メソッド・レスポンス形式が detailed-design.md と一致しているか
- テーブル名・カラム名・型が db-design.md の CREATE TABLE 文と一致しているか
- requirements.md の機能要件がすべて実装されているか（未実装は必ず指摘）
- 非機能要件（性能・セキュリティ・ログ）への対応が実装されているか

## セキュリティチェック観点
- SQLインジェクション対策（プレースホルダー使用）
- XSS 対策（出力エスケープ）
- 認証・認可の抜け漏れ
- センシティブ情報のハードコード（パスワード・APIキー）
- エラーメッセージに内部情報が含まれていないか

## 品質チェック観点
- エラーハンドリングが detailed-design.md の方針に従っているか
- ログ出力が適切か（過剰・不足どちらも指摘）
- 環境依存の設定値が環境変数・設定ファイルで管理されているか

## 指摘の重大度分類
- **重大**：セキュリティ脆弱性・必須機能の未実装・データ破壊リスク
- **中**：設計との乖離・エラーハンドリング不足
- **軽微**：コードスタイル・命名・コメント不足
