# 正規化ルール

## 第1正規形（1NF）
- 各カラムは原子値（配列・JSON を直接格納しない）
- ただし、検索不要な設定値はJSONカラムの使用を許容する

## 第2正規形（2NF）
- 複合主キーの場合、すべての非キー属性が主キー全体に従属すること

## 第3正規形（3NF）
- 非キー属性間の推移的従属を排除する
- 例：users テーブルに prefecture_name を持たない → prefectures テーブルを作り外部キーで参照

## 非正規化を許容するケース
以下の場合、パフォーマンスのために非正規化を検討できる。
ただし必ずカラムコメントに理由を記載すること。

- 集計値（注文総額等）：更新頻度が低く参照頻度が高い場合
- 住所の非正規化：住所変更時に履歴を保持したい場合
- 非同期で更新するキャッシュカラム
