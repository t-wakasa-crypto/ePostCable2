# 要件定義 品質基準

## 必須セクション（すべて含めること）
1. 背景・目的（なぜ作るか）
2. ユーザーストーリー形式の機能要件
3. 非機能要件
4. 制約条件
5. 用語定義
6. 未確認事項リスト

## ユーザーストーリーの書き方
「〇〇として、△△したい。なぜなら□□だから。」形式を使う。
詳細な例は `references/user-story-examples.md` を参照すること。

## 非機能要件のチェック
性能・セキュリティ・可用性・保守性・拡張性の5軸で確認する。
チェックリストは `references/non-functional-checklist.md` を参照すること。

## 品質チェックリスト
- [ ] 誰が使うか明確か
- [ ] 曖昧な表現（「高速に」「使いやすく」）を使っていないか
- [ ] 数値で測定できる基準があるか
- [ ] 矛盾する要件がないか
- [ ] 実装可能な範囲か
