# ユーザーストーリー 記述例

## 良い例

「管理者として、ユーザーの一覧を検索・絞り込みしたい。
なぜなら大量のユーザーの中から対象者を素早く特定するため。」

受け入れ条件：
- 名前・メールアドレス・登録日での検索ができる
- 検索結果は100件を超える場合にページネーションされる
- 検索結果のCSVエクスポートができる

## 悪い例（修正前）

「ユーザー管理機能がほしい」
→ 誰が・何を・なぜが不明。受け入れ条件もない。

## 粒度の目安

1ストーリーの開発工数目安：0.5〜3日程度。
それ以上になる場合はストーリーを分割すること。

## エピックとストーリーの関係

エピック（大機能）：認証機能
  ├── ストーリー：メールアドレスでログインしたい
  ├── ストーリー：パスワードをリセットしたい
  └── ストーリー：ログイン状態を14日間維持したい
