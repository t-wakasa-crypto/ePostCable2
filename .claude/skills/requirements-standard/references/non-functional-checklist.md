# 非機能要件 チェックリスト

## 性能要件
- [ ] レスポンスタイム目標（例：通常操作は2秒以内、検索は5秒以内）
- [ ] スループット（例：100リクエスト/秒）
- [ ] 同時接続ユーザー数
- [ ] データ量上限と増加見込み

## セキュリティ要件
- [ ] 認証方式（OAuth2 / SAML / ID+PW）
- [ ] 認可・権限管理の方式
- [ ] 通信暗号化（TLS1.2以上）
- [ ] ログイン試行回数制限・アカウントロック
- [ ] 個人情報の取り扱い（GDPR / 個人情報保護法）
- [ ] 脆弱性診断の要否

## 可用性・信頼性
- [ ] 稼働率目標（99.9% = 年間8.7時間以内の停止）
- [ ] 計画停止の許容時間帯
- [ ] バックアップ頻度とリストア目標時間（RTO/RPO）
- [ ] 障害発生時の通知方法

## 保守性・運用性
- [ ] ログの保存期間と形式
- [ ] 監視・アラートの要件
- [ ] デプロイ頻度と方式（Blue-Green / Canary等）
- [ ] 設定変更の反映方法（再起動要否）

## 拡張性・移植性
- [ ] ユーザー数・データ量増加時のスケール方針
- [ ] 将来連携が想定される外部サービス
- [ ] 対応ブラウザ・デバイス
