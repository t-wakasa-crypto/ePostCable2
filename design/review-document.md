## 1. システム概要

このシステムは、**お客様への請求書・納品書の作成からメール送付までを自動化する社内向けの管理システム**です。営業・カスタマー担当の方が日々の送付状況を画面で確認し、必要に応じて送り直しなどの是正を行えます。

これまで請求書・納品書は、作成 → PDF 化 → メール送付までを手作業で行っており、「送り忘れ」「宛先の間違い」「誰にいつ送ったか記録が残らない」「大量処理に手間がかかる」といった課題がありました。本システムは、基幹システムの出荷データをもとにこれらの作業を自動で行い、送付漏れ・誤送信を防ぎ、送付の状況をいつでも画面で追跡・是正できる状態にすることを目的としています。

## 2. 主要機能一覧

利用者から見て、本システムでできることは以下のとおりです。

| 機能 | 何ができるか |
|---|---|
| 自動での書類作成 | 基幹システムに登録された出荷情報をもとに、請求書・納品書を毎日自動で作成します。作業者の操作は不要です。 |
| 自動でのメール送付 | 作成された書類を、深夜のうちに自動でお客様へメール送付します（PDF を添付）。 |
| 送付状況の一覧確認 | 請求書・納品書ごとに「送付前・送付中・送付済み・失敗」といった状況を一覧で確認できます。 |
| ダッシュボード（全体状況の把握） | 送付の全体像（件数の内訳、直近の実行結果など）をトップ画面で一目で確認できます。 |
| 手動での送り直し | 送付に失敗した書類を、画面のボタンから手動で送り直せます。 |
| 宛先メールアドレスの修正 | 送付に失敗した書類の宛先メールアドレスを修正できます（誤送信防止のため管理者のみが操作可能）。 |
| まとめて送り直し | 失敗した複数の書類を、まとめて再送付の対象に戻せます（管理者のみ）。 |
| PDF・CSV のダウンロード | 書類の PDF や、一覧情報を CSV ファイル（Excel などで開ける形式）としてダウンロードできます。 |
| 送付履歴の確認 | いつ・どの書類を・誰に送ったか、また失敗した理由を含めて履歴を追跡できます。 |
| 出荷取り込み履歴の確認 | 基幹システムからの取り込みが正常に行われたかを履歴で確認できます。 |
| 利用者アカウントの管理 | 利用者の追加・変更・退職処理などを行えます（管理者のみ）。 |
| 動作設定の変更 | 再送付の回数や通知先メールアドレスなどを画面から変更できます（管理者のみ）。 |
| ログイン・権限管理 | メールアドレスとパスワードでログインします。担当者向け画面と管理者専用画面が分かれています。 |

### 利用者の種類（権限）

| 利用者の種類 | できること |
|---|---|
| 一般担当者（営業・カスタマー担当） | 送付状況の確認、手動での送り直し、PDF・CSV のダウンロード |
| 管理者 | 上記すべてに加え、取り込み・送付の手動実行、まとめて送り直し、宛先メールアドレスの修正、利用者管理、動作設定の変更 |
| 退職者 | ログインできません（在職中のアカウント情報が残っていても利用不可） |

## 3. 利用シナリオ（代表的な使い方）

### シナリオ 1: 担当者が送付状況を確認したいとき

1. ログイン画面でメールアドレスとパスワードを入力してログインする
2. トップ画面（ダッシュボード）で、全体の送付状況（件数の内訳・直近の実行結果）を確認する
3. 詳しく見たい場合は「請求書一覧」または「納品書一覧」を開く
4. 状況（送付前・送付中・送付済み・失敗）で絞り込んで、対象の書類を探す
5. 個別の書類を開くと、明細と送付履歴（いつ・誰に送ったか）を確認できる

### シナリオ 2: 担当者が送付に失敗した書類を送り直したいとき

1. 「請求書一覧」で状況「失敗」を選んで、失敗した書類を表示する
2. 対象の書類を開く
3. 失敗した理由を確認する（例: 宛先のメールアドレスが正しくない、など）
4. 「送り直し」ボタンを押す
5. システムが再度メール送付を行い、状況が更新される

### シナリオ 3: 管理者が宛先メールアドレスを修正して送り直したいとき

1. 管理者でログインする
2. 送付に失敗した書類（宛先アドレスが誤っているもの）を開く
3. 「メールアドレス編集」で正しい宛先に修正して保存する（この操作は管理者のみ可能）
4. 「送り直し」またはまとめて再送付の対象に戻す操作を行う
5. 正しい宛先へ再度メールが送付される

### シナリオ 4: 管理者が動作設定を変更したいとき

1. 管理者でログインする
2. 「システム設定」画面を開く
3. 再送付の回数の上限や、バッチ完了通知の送り先メールアドレスなどを変更して保存する
4. 必要に応じて「テストメール送信」で、指定したアドレスへ実際にメールが届くか確認する

### シナリオ 5: 管理者が取り込み・送付を手動で実行したいとき（自動処理が失敗した場合の救済）

1. 管理者でログインする
2. 「出荷取り込み履歴」画面を開き、自動取り込みが失敗していないか確認する
3. 失敗していた場合は「取り込み実行」ボタンから手動で取り込みを実行する
4. その後、必要に応じて請求書・納品書の一覧画面から送付を手動で実行する

## 4. 制約・前提条件（あらかじめご了承いただきたい点）

利用者の皆さまに知っておいていただきたい、システムの制限事項です。

### 4-1. 金額・明細は編集できません

- 請求書・納品書の**金額や明細（品目・数量・単価など）は、画面から編集できません**。これらは基幹システムで確定した売上データをそのまま使用します。
- この仕様は**将来的にも変更しません**。金額を修正する機能は追加されない前提です。
- 修正できるのは、宛先メールアドレス（送付に失敗した書類のみ・管理者操作）です。
- 万一、金額データに不整合が見つかった場合は、当面はシステム管理者がデータを直接修正して対応します。

### 4-2. 自動処理のスケジュール

- 出荷データの取り込みは**毎日昼（12:15〜12:30）**に自動実行します（基幹システムの請求データ確定が正午のため、その後に実行）。
- 請求書・納品書のメール送付は**毎日深夜**に自動実行します（納品書は 01:00、請求書は 01:30）。
- 送付に失敗した書類のまとめて再送付は**毎週月曜の深夜**に自動実行します。
- 自動処理が失敗した場合は、管理者が画面から手動で実行して救済できます。

### 4-3. 送付に失敗したときの扱い

- 一時的な原因（メールサーバの一時的な不調など）による失敗は、**自動で数回まで送り直し**を試みます（回数は設定変更可能です）。
- 宛先が明らかに不正な場合（宛先が1件も無い、メールアドレスの形式が正しくない）は、**自動での送り直しは行わず**、管理者による宛先修正などの手動対応が必要です。
- まとめて送り直しを行う際、すでに3回以上失敗している書類が含まれる場合は確認メッセージを表示します。10回以上失敗しているものは、まとめて送り直しの対象からは除外されます。
- 送付に失敗したままの履歴は削除されず記録として残りますが、通常の一覧・ダッシュボードには表示されません（専用の確認画面でのみ確認できます）。

### 4-4. 送付先メールアドレス

- 1つの書類につき、**送付先は最大3件**まで設定できます。
- 宛先アドレスの修正は、送付に失敗した書類に対してのみ、管理者が行えます（誤送信のリスクを避けるため）。

### 4-5. 利用環境・運用に関する前提

- 対応ブラウザは**最新の一般的なブラウザ**です（古いブラウザには対応しません）。
- 同時に利用する人数は**10人程度**を想定した社内システムです。
- 画面の表示は**3秒以内**を目標としています。
- 稼働率の目標は**平日日中で99%**です。
- ログ（履歴）は**7年間**保管します。
- お客様情報（個人情報）は社内規則に従って取り扱います。通信は暗号化されます。
- 万一の障害からの復旧目標は**翌営業日以内**です。
- 本システムは基幹システムおよび社内データベースと連携しているため、それらが停止している間は本システムも利用できません。
